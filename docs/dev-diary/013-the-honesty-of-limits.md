# 開発日記 013: 誠実な限界 — 透明性の UI

## 概要
収益化はプロダクトの継続性のために必要不可欠だが、それをユーザーに強いるのではなく「納得」してもらうことが重要である。本日、未ログイン時の機能制限を明確化し、ログイン済みの無料ユーザーに対して「本日の残り回数」を解析ボタンの直近に提示する UI を実装した。

## 議論と洞察のポイント

### 1. 期待値のコントロール
ユーザーがボタンを押した後に「回数制限です」と告げられるのは、不快な体験である。事前に「あと何回使えるか」を提示することは、誠実なインターフェースの証である。3 回という無料枠の限界を隠さず、かつ美しく表示することで、ユーザーは自身の利用計画を立てることができ、プレミアムプランの価値もより自然に伝わるようになる。

### 2. 自由な場所としてのポータル
「解約方法がわからない」という不安は、課金そのものへの心理的障壁となる。Stripe のカスタマーポータルへの直接的な導線を引くことで、ユーザーは「いつでも自分でコントロールできる」という安心感を得る。Local-First な哲学を掲げる以上、データの主権だけでなく、支払いの主権もユーザーに属すべきである。

### 3. 未登録時の「閉ざし方」
解析機能にはログインを必須としたが、その理由は「無料枠を公平に管理するため」という一文を添えるようにした。単に「ログインしてください」と拒絶するのではなく、理由を添えてサインアップという一歩を促す。この対話の積み重ねが、プロダクトへの信頼を築く。

## 結論
制限を可視化することは、自由を奪うことではなく、信頼のベースラインを引くことである。本日の透明性向上により、Nutri-Vision は健全な収益化モデルへの第一歩を力強く踏み出した。

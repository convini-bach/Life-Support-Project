# 開発日記 012: 構造的整合性 — 設定画面の再設計

## 概要
設定画面（プロフィール）は、プロダクトの「心臓部」である。本日、私たちはこの画面の構造を根本から見直した。AI エンジンの高度な設定（API キー、モデル選択）と、ユーザーの基本設定（言語、目標）を物理的・視覚的に分離し、開発者モード購入者にのみ必要な情報を開示する条件付きレンダリングを導入した。

## 議論と洞察のポイント

### 1. 役割に応じた情報の出し分け
すべてのユーザーにすべての設定項目を提示することは、親切ではなく「ノイズ」である。一般ユーザーにはシンプルな身体データ設定を、自分の API キーを使いたいパワーユーザー（開発者モード）には詳細な AI 構成を。この「情報の階層化」により、画面はより軽やかになり、ユーザーは自分の役割に集中できるようになった。

### 2. 目標入力欄の「記入例」という小さな配慮
「分析時の優先事項」という自由入力欄に、「体重を減らしたい、塩分を控えたい」といった具体的な記入例（プレースホルダー）を提示した。自由度は、時としてユーザーを立ち止まらせる。小さなガイドを置くことで、ユーザーの筆を動かし、AI が提供するアドバイスの質を最大化する土壌を整えた。

### 3. セクションタイトルの動的な躍動
Nutri-Vision 画面において、選択されているタブ（食事・運動・体重）に合わせて履歴セクションのタイトルが動的に変化するようにした。「記録」という抽象的な言葉から、「本日の食事記録」という具体的な言葉へ。UI が状況を理解していることを示すことで、ユーザーとの対話性が深まる。

## 結論
情報の整理は、ユーザーの認知リソースを節約するための礼儀である。本日の構造改革により、Nutri-Vision プロジェクトはより高度で、かつ親切な「道具」へと進化した。
